Where the type system can make a wrong call impossible to construct, do that instead of accepting a wide type and validating it at runtime — a caller should not be able to compile a call that fails.
_Avoid_: a runtime `if`-chain re-checking a constraint the type could have enforced at the boundary; accepting `string` where only a closed set of values is legal; a blanket-optional input — `Partial<T>` or a record with every field `?` — for an operation that requires a definite subset present — spell the per-operation input type instead.
Detect: a runtime guard clause at the top of a function whose only job is rejecting inputs the parameter type allowed through; a validation function whose output is discarded (its boolean is checked, not its narrowed value threaded onward); a `Partial<T>` or all-fields-optional record accepted as a domain/application input and guarded field-by-field inside.
Not-when: the constraint depends on data unavailable at compile time (a DB lookup, a cross-field business rule checked against live state) — that stays a runtime `Result`, not a branded type.
