## The criterion (locked via grilling, #149)

Three delivery vehicles exist (#150): a pathless always-loaded **digest line**, an on-demand **body**, and a surviving **`paths`-rule**. Tier is decided by:

1. **Surface hard-gates the digest.** The digest is pathless and always-loaded, so a surface-gated law placed in it re-injects on every unrelated `.ts` edit — the exact cost this map kills. Therefore **`core` is drawn only from `when:always` rules.** All 28 surface-gated judgeable rules are `pull-only` *by construction*.
2. **`core` = delivery-need, not enforcement class.** A rule is `core` if it is always-surface, high-value, AND Claude demonstrably does not do it unprompted. This admitted 2 **deterministic** always-rules (`branded-primitives`, `exhaustiveness`) as **interim tenants** — they have no linter yet (#152 unbuilt), so the digest is their only delivery. When #152 ships, they are hoisted out atomically. (`exhaustiveness` left early — demoted by measurement at #154, below — leaving `branded-primitives` the sole tenant.)
3. **`core` is an empirical hypothesis, not a taste call.** "Claude doesn't do it unprompted" is measurable — so **#150's compliance eval is the labeler of record for `core`**, per `correctness-is-a-measured-rate`. The 21 below (post-#154) are provisional; the eval promotes/demotes members against measured digest-only-vs-full violation deltas. The **`opt-in`** set, by contrast, is a taste/content-policy call and is labeled by the human (locked, below).

## The opt-in gate (deterministic, zero-LLM)

Realized by **physical corpus split**, not a model-read flag (`paths` is the only reliably-recalled frontmatter key):

- `sound/corpus/` — standard rules, default-install.
- `sound/corpus-optin/` — the 3 opt-in rules; **not** in Phase 2's candidate universe.
- `sound:setup` Phase 2 globs `corpus/` only. An opt-in rule becomes a candidate **only** when the user explicitly names it. Directory membership *is* the gate — no judgment, no frontmatter recall.
- `score.mjs` / fixtures: the valid-name universe becomes `corpus/ ∪ corpus-optin/`; the default-install universe stays `corpus/`. A fixture that doesn't opt in must not list an opt-in rule in its expected set.
